# Current-target open RED inventory

This record inventories independent correctness roots that remain RED on the Rolldown pull request #10387 preview (`dce23e36a`, preview entry SHA-256 `b369e2205adb6c1f6f648509c5498a10a0847c6a87c2c090f88d35014ef0f88c`). The investigation started from fuzzer head `0047c2a`; the resulting minimal controls and current-target audit landed as the follow-up described below. It follows the release-gap audit but counts product mechanisms rather than generated cases, seeds, signatures, wrap modes, or implementation sites.

Family A is the only confirmed strict-SEO-only correctness root after #10387. Its flag-off control was checked against actual source and bundle events, not merely the relaxed oracle: values and phase markers agree, and the only difference is a legal within-phase reorder. Its current bundle leaves the pure definer uninitialized while a namespace consumer observes that binding. The exact shadcn transplant overlay is byte-identical to the model used for the #10104 closure campaign. The frozen `42628c18b` build was recorded GREEN, but the published #10104 merge artifact `1d86ffe` is RED in both wrap modes; therefore this was already present at merge head rather than introduced by #10322 or a later main commit. The broad interval includes the late `preserve tree-shaking across strict modes` work, but a first-bad attribution still requires a build bracket and must not be inferred only from the source diff.

The CJS wrapper-cycle root is represented by one parameterized two-module builder: one CJS entry requires one leaf, a manual group contains only the entry, `includeDependenciesRecursively` is false, and output is CJS. A CJS leaf exercises `require_module_*`; changing only the leaf format to ESM exercises `init_module_*`. The group chunk requires the entry facade and calls the exported wrapper before the facade has installed its getter. ESM output, automatic chunking, or recursive dependency inclusion passes. The larger organic and minified random witnesses add no implementation path and are not retained.

## Full-fuzzer interpretation

The complete 100-case-per-cell current-preview sweep produced identical on-demand and wrap-all totals: mixed `22/100`, pure ESM `24/100`, and pure CJS `2/100`. Every ESM RED is Family A: ordinary inferred-pure and callable-own-state variants produce the NaN/primitive-value crash, while three packaged variants produce an event reorder. The two pure-CJS REDs are the #9998 cross-entry leak arm and the CJS wrapper-cycle root. A 300-case-per-cell extension was stopped after five cells rather than spending more release time: mixed was `65/300` in both wrap modes, pure ESM was `61/300` in both modes, and pure-CJS on-demand was `4/300`; targeted replay and shrinking of the additional failures found no fifth root. Because the sixth cell did not finish, there is intentionally no final 300-cell evidence file and the complete 100-cell matrix remains the reproducible sweep result.

The existing directed current-preview matrix is otherwise green: #10322 class forms, the adjacent eager analyzer surface including the computed object key, #10336 and adjacent deconfliction controls, cross-chunk init cycle, entries-aware manual-chunk cycle, optimizer runtime-placement/facade cycle, the older CJS-output witness matrix, transpiled-CJS interop, dynamic-entry/wrap-kind merge, exotic import reads, and Family B. The release registry is `69/72` required PASS solely because the three manual-pure cases remain RED; that narrow count does not include the standing random and transplant surfaces above.

## What is not another correctness root

The 934 built-in/prototype mutation probes are one optimizer-assumption surface, not 934 correctness bugs. Getter and `Proxy` effects under `propertyReadSideEffects:false` are the caller-selected trade-off. #10294 output-shape and metadata leads are outside the fuzzer's source-versus-bundle runtime oracle and remain unconfirmed until a semantic or API-level witness exists.

## Coverage without test inflation

Family A already has generated coverage, a shrunk five-module history, and three unchanged real-graph transplant overlays; no syntax-equivalent fixture was added. The gap was execution policy: the current-target release run exercised only the new #10322/#10336 adjacency registry, while `reacceptance.ts` deliberately accepts broad reason classes against its frozen buggy target. `npm run audit:current-target` now reuses pure-ESM seed `200004`, verifies its `strictExecutionOrder:false` control, and reports each directed root separately. It accepts either the exact known-root signature or a future PASS; a different RED fails, so another `bundle-only-crash` cannot hide inside the historical reason bucket.

The current-target audit includes exactly one on-demand CJS-output strict control for #9998 because it reaches an output arm the historical ESM campaign did not; wrap-all has the identical signature and is not duplicated. The historical campaign and its evidence schema remain unchanged. The CJS wrapper cycle keeps exactly two two-module cases, one per lowering arm: a CJS target exercises `require_module_*`, while an ESM target exercises `init_module_*`. The current-target audit on the #10387 preview reports four known RED rows—Family A, CJS strict cross-entry leak, and the two facade lowerings—plus the passing Family-A flag-off control. The three manual-pure cases remain useful despite sharing one product root because they cover two distinct analyzer repair sites, including the `NewExpression` callee path that a member-only fix cannot reach; they remain owned by the release registry and are not copied into this audit.
