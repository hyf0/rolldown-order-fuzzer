# Release gap audit for Rolldown #10320, #10322, and #10336

This audit was performed before the 2026-07-22 release. The fuzzer baseline was `6249844207f614ba35fbc4559af3ffdd47d801ad`; the measured Rolldown main build was `aa73a684830e9f0d024795e2294c5d075c2ed020`, including Oxc 0.141.0. The tested CI artifact came from run `29886984898`; its resolved entry SHA-256 was `5e4bb0e65697dda7280d66a0c3263f6c94cdd9aa5cba5b67cf1a5bad2fb3d71e`. This work extends the differential semantic oracle described in [the MVP](./execution-order-fuzzer-mvp.md) and adds a maintained directed release gate alongside the broader seeded campaign.

## #10320 is outside the semantic oracle

#10320 did not fix a source-versus-bundle correctness failure. One change sorted strict-mode transitive init registration so generated symbol suffixes are deterministic across 32-bit and 64-bit targets; the other replaced a linear membership scan with an equivalent constant-time lookup. The PR reported no behavior or snapshot changes.

The fuzzer runs one Rolldown target and compares source and bundle execution. It intentionally does not compare output bytes across architectures and is not a performance benchmark. #10320 therefore has no witness for this oracle. Cross-target reproducibility and performance need dedicated tests rather than a weaker source-versus-bundle assertion here.

## Why #10322 previously had zero probability

The old generator had no event-free class definition that read mutable global state at definition time and no separate downstream observer. Adding an ordinary event to the reader would itself make the module visibly order-sensitive and could hide the missing class analysis, so no seed could accidentally assemble the necessary patch, reader, and observer topology.

The new analyzer witness does not modify a standard built-in. The patch module installs a fixture-owned pure function, `globalThis.__orderRead = () => expected`; the reader evaluates `(/* @__PURE__ */ globalThis.__orderRead?.() ?? fallback)` through one selected syntax form; and a downstream observer imports the patch before the reader and then reads the exported result. The annotation truthfully states that calling the fixture function has no side effect. It does not state that the function exists before the earlier patch or that its returned value is invariant across execution order.

The verified #10322 bracket contains eight forms: `class-static-field-declaration`, `class-static-field-expression`, `class-static-field-default-export`, `class-static-field-iife`, `class-heritage`, `class-computed-key`, `class-computed-accessor-key`, and `class-nested-static-field`. They are 8/8 RED on the audited pre-fix build and 8/8 PASS on the fixed build. The last three added witnesses independently cover the PR's direct-IIFE special path, computed `MethodDefinition` key, and recursive nested-class visitor rather than treating one static `PropertyDefinition` as representative of all three. `class-static-block` and `direct` remain green boundary controls: making a static-block result observable requires an assignment that already makes the class effectful in the pre-fix analyzer.

## Why #10336 previously had zero probability

The old model could not request the actual `output.codeSplitting: false` build that co-hosts Rolldown's runtime and author code in one root scope. The renderer also minted collision-avoiding local bindings instead of accepting an exact author-selected name, and `generatedCode.profilerNames` was fixed off. It therefore could not put an authored runtime-helper-like name next to the force-included runtime helper.

The new witness uses strict wrap-all, `codeSplitting: false`, a typed author-selected export binding, and a later wrapped module that consumes the helper. The exact #10336 names are `__esmMin` with profiler names off and `__esm` / `__getOwnPropNames` with profiler names on, each under ESM and CJS output. The unfixed base is 6/6 RED and the fixed build is 6/6 PASS. `__commonJSMin`, `__commonJS`, and `init_module_0002` stay as adjacent green deconfliction controls.

## Capabilities added by the audit

- A canonical set of 61 global-read AST forms covering direct reads, class definition-time work, immediately invoked functions, array and object expressions, destructuring, assignments, pure annotations, manually pure functions, construction, instance fields, default parameters, returned classes, and three manual-pure child-effect regressions.
- A typed fixture-owned global function assignment and optional-call reader for the 57 analyzer forms that do not depend on built-in mutation.
- `authoredExportBindings`, a typed way to request an exact source binding name without accepting arbitrary source text.
- `Chunking.kind: "disabled"`, mapped to actual `output.codeSplitting: false`, plus persisted `profilerNames`, property-read/write side-effect options, and manual-pure functions.
- A call-count observation channel for effect-preservation probes whose final value does not reveal whether an optimizer deleted evaluation.
- Closed typed registries for 45 mutable built-in calls, four direct `instanceof` expressions, and 884 additional optimizer expressions. Models persist reviewed keys rather than arbitrary executable source.
- A case-owned wrapping mode so the #10336 generator always requests wrap-all while analyzer cases request on-demand wrapping.
- Runner intrinsics captured before fixture execution so replacing built-ins cannot poison error normalization or event validation.
- A durable model fingerprint for every reserved release-gap corpus replacement.

The ordinary seeded campaign reserves only lane 0 for the 57 analyzer forms and lanes 1 and 6 for ESM and CJS authored-name cases. Optimizer assumption probes remain fully represented in the directed registry but no longer consume 16 of every 32 ordinary random seeds.

## Required analyzer findings

The fixture-owned function matrix proves that the adjacent analyzer failures do not rely on monkey-patching. On `aa73a6848`, all 57 analyzer forms had valid source and bundle runs: ten passed and 47 produced an event-value mismatch. The passing forms were `direct`, the eight fixed #10322 class forms, and `class-static-block`. The 47 failures span IIFEs, array/object wrappers, binding patterns, assignment, annotated/manual-pure paths, construction, instance fields, default parameters, and returned classes.

Every failing source evaluates the patch first and observes the fixture function's returned value. The bundle evaluates the unwrapped reader chunk first and records the fallback. Since the function and global property belong to the fixture and the pure-call annotation is truthful, these are strict execution-order analyzer/wrapping correctness failures rather than optimizer-assumption violations.

The 47 failures divide into 15 immediately invoked function forms whose bodies are not visited by the general call analyzer, 15 outer member-expression forms whose object/computed-key order reasons are not propagated, four binding-pattern forms whose defaults or computed keys are skipped, one side-effect-free assignment arm that skips its RHS, and 12 manual-pure call/new forms whose tree-shaking purity is not translated into an order reason. All 47 execute the relevant body, initializer, or constructor synchronously at module evaluation time; none is a legitimately deferred function body, method body, or unconstructed instance initializer.

The robust Rolldown fix is to generalize #10322's definition-time order-reason collector into a top-level eager-evaluation collector instead of adding another tree-shaking exception for each wrapper. It should collect only order reasons, independently of Oxc's tree-shaking early exits, while retaining the existing deferred boundaries. This leaves `tree_shaking_flags`, DCE, inline constants, pure annotations, manual-pure functions, and property read/write settings unchanged. The intended cost is limited to strict on-demand builds: modules that really depend on earlier observable state receive a wrapper and are no longer sorted as unordered pure leaves.

The array-length form is intentionally not part of this required group. `[call()].length` always has value `1`; only a separate call counter can detect whether the element call ran. Observing a side effect after declaring the fixture call pure would contradict the annotation, so the maintained effect probe instead uses the existing `Math.hypot` monkey-patch and belongs to the assumption group.

## Required manual-pure child effects

The audit also found a separate ordinary tree-shaking correctness bug that does not depend on strict execution order. `treeshake.manualPureFunctions: ["make"]` permits Rolldown to delete an unused call to `make`, but JavaScript still requires eager side effects in that call's arguments and in computed keys evaluated around its result. On `aa73a6848`, each of the following source shapes increments a fixture-owned counter to one while the bundle leaves it at zero:

```js
make()[effect()];
make(effect()).value;
new (make()[effect()].Box)();
```

The three maintained required cases cover separate analyzer paths: a member-expression computed key, a call argument inside a member-expression object, and a computed key inside a `new` expression's callee. The last case matters because repairing ordinary member-child propagation does not by itself make `NewExpression` visit its callee. No built-in is modified, and `effect` is not declared pure.

This classification does not require Rolldown to preserve every behavior reachable behind a manually pure call. Effects that occur only by invoking a getter, method body, or selected constructor from the returned value remain assumption/trade-off probes. The required boundary is narrower: eager argument, computed-key, and callee evaluation that JavaScript performs outside the declared-pure function body must survive deletion of the pure call.

## Optimizer assumption probes

The expanded optimizer registry contains 934 unique probes: one array-length effect case, 45 direct built-in calls, four `instanceof` expressions, and 884 additional expressions. On `aa73a6848`, all 934 produce a source-versus-bundle semantic mismatch, and every bundle result equals the result expected if the standard built-in had not been modified.

All 934 probes deliberately modify a standard built-in object, prototype, constructor, or global binding: 200 modify static/global built-in properties, 667 modify built-in prototypes, and 67 replace constructors or global bindings. There are 899 value differences and 35 deleted-call observations. Nine run with `minify: true`; 925 run with `minify: false` and reach Rolldown's default DCE or inline-constant paths.

Oxc explicitly documents “No Monkey-Patching Built-ins” as an unconditional minifier assumption in `crates/oxc_minifier/docs/ASSUMPTIONS.md`. That directly classifies the nine minified probes as expected trade-off checks. Rolldown currently does not publicly state whether its default non-minify DCE and inline-constant evaluation adopt the same assumption, so the remaining 925 expose a product-policy and documentation gap rather than 925 established correctness bugs. The probe policy is deliberately neutral: keep measuring the boundary, do not block this release on its mismatches, and reclassify only after Rolldown states a different semantic contract.

These probes contain no input inside the no-monkey-patching assumption, so they cannot prove that optimizer behavior is correct within the assumption. They also must not be reported as 934 independent root causes.

The `optimizer-array-length-call-effect` metadata now records the rendered expression's actual fallback value, `1`, rather than `Math.hypot`'s unused return value `5`. Equal expected/fallback values are accepted only when validation proves the full patch → expression → counter observation topology.

The registry separates required semantic cases from non-blocking assumption probes:

| `adjacent/deconfliction`    | required         |          3 |                                  0 |                           0 |

The 49 adjacent analyzer cases include the 47 current catches plus the green `direct` and `class-static-block` boundary controls. The array-length effect case is registered exactly once in the assumption group. A semantic mismatch from an assumption probe is recorded as an observation and does not change the exit code; invalid source, build failure, or harness failure still rejects the gate because those outcomes mean the probe did not run correctly.

Remote main advanced to `c07a55cd598857d0e54d0d2083142cf3fa184ed6` after the full `aa73a6848` audit. Its pkg.pr.new snapshot was therefore rerun through all 69 required cases: 50 remained RED and 19 passed, exactly matching the table's required split. The six intervening commits include #10320 plus test, runtime-lifecycle, and ESM-metadata changes; none changed these semantic findings. The resolved `rolldown@1.2.0` entry SHA-256 was `b3b9b02c4eaa49d845740635b4d1537dcfc8ff262afc96f1b01c57911cee8487`.

Run the maintained gate against the exact release candidate:

```sh
RELEASE_ROLLDOWN=/absolute/path/to/rolldown/dist/index.mjs npm run release:order-regressions
```

Case ids may follow `--` to run a subset. Machine-readable evidence is written to `.agents/evidence/release-order-regressions.json` by default; set `EVIDENCE_OUT` when preserving multiple target runs.
